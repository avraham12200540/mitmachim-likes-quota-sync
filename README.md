# Mitmachim Likes Quota Sync

תוסף Chrome (Manifest V3) שמציג **מד לייקים יומי מסונכרן בין מחשבים** עבור הפורום
[mitmachim.top](https://mitmachim.top). המד מוצג כפס אנכי בצד שמאל של המסך,
מתמלא מלמטה למעלה, ומציג מספר כמו `13/20`. הספירה נשמרת בשרת (מודול שנוסף
לפרויקט ExtSync הקיים), כך שאותו משתמש רואה את אותו מצב בכל מחשב.

> התוסף **לא** עוקף מגבלות, לא מבצע לייקים אוטומטיים ולא משנה הרשאות. הוא רק
> מאזין לפעולות שהמשתמש עושה בעצמו, סופר אותן, ומסנכרן את הספירה.

---

## מבנה

```
extension/
  manifest.json            # MV3, הרשאות מינימליות (storage + host)
  content/
    config.js              # כל ההגדרות וה-selectors במקום אחד
    user.js                # זיהוי המשתמש המחובר (NodeBB app.user)
    page-probe.js          # רץ ב-main world של העמוד, קורא את window.app
    api.js                 # לקוח דק שמדבר עם ה-service worker
    detector.js            # זיהוי לחיצות לייק / ביטול לייק
    widget.js              # רכיב התצוגה (המד)
    content.js             # orchestrator - מחבר הכל יחד
    styles.css             # עיצוב המד
  popup/
    popup.html / popup.css / popup.js   # ניהול ידני
  background/
    service-worker.js      # שער ה-API היחיד + cache + תור אירועים
  icons/                   # icon16/48/128
```

### למה כל קריאות הרשת עוברות דרך ה-service worker?

ב-Manifest V3 קריאת `fetch` מתוך content script כפופה ל-CORS של העמוד. ה-service
worker, בזכות `host_permissions`, יכול לפנות ל-API ב-cross-origin בלי בעיה. לכן
ה-content script וה-popup שולחים הודעות (`chrome.runtime.sendMessage`) וה-service
worker מבצע את ה-fetch בפועל. יתרון נוסף: הטוקן נשמר אך ורק בצד ה-service worker
ולא נחשף בקוד שרץ בתוך העמוד.

ה-`chrome.storage.local` משמש כ"פס סנכרון" בין ה-popup ל-content script: כל שינוי
ב-cache משדר אוטומטית (`storage.onChanged`) וכל הצדדים מתעדכנים מיד.

## טעינה (Unpacked)

1. פותחים את `chrome://extensions`.
2. מפעילים **Developer mode** (פינה ימנית עליונה).
3. **Load unpacked** ובוחרים את תיקיית `extension/`.
4. נכנסים ל-`https://mitmachim.top` ומתחברים לחשבון.

## זיהוי משתמש (forum login) - בלי הגדרות

ברירת המחדל: התוסף מזהה אותך אוטומטית לפי **ההתחברות שלך ל-mitmachim.top**. ה-
service worker קורא את ה-session cookie של הפורום ושולח אותו לשרת, שמאמת אותו מול
NodeBB (`/api/self`) ומקשר את המכסה ל-uid המאומת. כלומר: מי שמחובר לפורום מקבל
סנכרון **בלי טוקן ובלי שום הגדרה**. אותו משתמש פורום = אותה ספירה בכל מחשב.

## כתובת API / טוקן (אופציונלי)

ברירת המחדל של כתובת ה-API היא `https://api.extsync.com`. אפשר לשנות בלי לבנות
מחדש את התוסף - אייקון התוסף ← **הגדרות מתקדמות**:

1. **כתובת שרת ה-API** - למשל `http://localhost:8000` לפיתוח.
2. **טוקן הזדהות (Bearer)** - *אופציונלי*. fallback לזיהוי כשאינך מחובר לפורום,
   או לשימוש אדמין. אם קיים גם cookie של הפורום - זהות הפורום גוברת.
3. **משתמש פיתוח (DEV ONLY)** - לשרת פיתוח עם `LIKES_QUOTA_DEV_AUTH=true`. ריק בפרודקשן.

לחלופין, אפשר לערוך ברירות מחדל ב-`content/config.js` ו-`background/service-worker.js`.

## בדיקות

### בדיקה ידנית מהירה
1. **לא מחובר לפורום** - המד מוצג אפור/לא פעיל (או מוסתר במסך צר).
2. **מחובר** - המד מציג את מצב היום, למשל `0/20`.
3. **לייק אחד** - לוחצים לייק על פוסט; אחרי ~0.7 שניות המד עולה ל-`1/20`.
4. **ביטול לייק** - מבטלים את הלייק; המד יורד חזרה.
5. **מעבר בין נושאים** - ניווט בפורום (NodeBB ajaxify); המד נשאר ולא מוכפל.
6. **גלילה ארוכה** - המד קבוע במקומו ולא מפריע לגלילה.
7. **popup** - לחיצה על האייקון מציגה מצב, נותרו, ורשימת המשתמשים שקיבלו לייקים.
8. **הגדר ל-20** - המד והפופאפ מתעדכנים מיד ל-`20/20`.
9. **איפוס ל-0** - חוזר ל-`0/20` והרשימה מתרוקנת.
10. **כשל שרת** - מכבים את השרת; המד מציג את הנתון האחרון מה-cache עם סימון
    "לא מסונכרן" (נקודה כתומה). מדליקים שוב; הספירה מסתנכרנת.
11. **רענון עמוד** - המד נטען מחדש עם המצב הנכון מהשרת.
12. **שני מחשבים / שני דפדפנים** - ראו למטה.

### בדיקת increment
- לוחצים לייק על פוסט של משתמש מסוים.
- בודקים שהמד עלה ב-1 ושב-popup מופיע אותו משתמש עם `1/6`.

### בדיקת decrement
- מבטלים את אותו לייק; המד יורד ב-1 והמשתמש יורד/נעלם מהרשימה.

### בדיקת popup set/reset
- "הגדר ל-20" ← `20/20`; "איפוס ל-0" ← `0/20`; שדה ידני (0-20) + "שמור".

### סימולציית "שני מחשבים"
1. דפדפן A ודפדפן B, שניהם עם אותו **טוקן** (אותו משתמש מאומת) מול אותו שרת.
   (אפשר גם שני פרופילי Chrome, או חלון רגיל + חלון אורח, כל אחד עם התוסף.)
2. ב-A נותנים 3 לייקים ← `3/20`.
3. ב-B מרעננים או מחכים עד 15 שניות (polling) ← אמור להציג `3/20`.
4. ב-B מבצעים "איפוס ל-0"; ב-A תוך 15 שניות (או במעבר פוקוס) המד יציג `0/20`.

> **חשוב:** הסנכרון "בין מחשבים" עובד כל עוד שני המחשבים מזדהים מול השרת כאותו
> משתמש (אותו טוקן ExtSync). השרת מקשר את המכסה למשתמש המאומת, ולא ל-uid
> שמגיע מהדפדפן - כך אי אפשר לזייף ספירה של משתמש אחר.

### בדיקת איפוס יומי
- השרת מחשב את "היום" לפי שעון ישראל (`Asia/Jerusalem`). אחרי חצות (שעון ישראל)
  בקשת `today` הראשונה מחזירה יום חדש עם `0/20`. לבדיקה מהירה אפשר לשנות זמנית את
  `LIKES_QUOTA_TIMEZONE` בשרת, או לבדוק סביב חצות.

### בדיקה שאין widget כפול
- מנווטים בין כמה נושאים ומדפי פרופיל ובחזרה; בודקים ב-DevTools
  ש-`document.querySelectorAll('#mt-likes-quota-widget').length === 1`.

### לוגים (debug)
ברירת המחדל שקטה. להפעלת לוגים מפורטים, ב-DevTools של הדף או ב-service worker:
```js
chrome.storage.local.set({ MTLQ_DEBUG: true });
```

## נקודות שצריך להתאים אם ה-markup של מתמחים טופ שונה בפועל

כל ה-selectors מרוכזים ב-[`content/config.js`](content/config.js) תחת
`config.selectors`. אם הזיהוי לא עובד, התאימו שם (ולא בקוד הלוגיקה):

| מה לבדוק | היכן | איך לאמת בפועל |
|----------|------|----------------|
| **כפתור הלייק** | `selectors.likeButton` | ב-DevTools, לחצו לחיצה ימנית על כפתור הלייק ← Inspect, וראו את ה-`component`/`class`/`data-action`. הברירת מחדל מכוונת ל-`[component="post/upvote"]` של NodeBB. |
| **אלמנט הפוסט** | `selectors.post` | מאיזה אלמנט אב נלקח ה-`data-pid`. בדרך כלל `[data-pid]` או `[component="post"]`. |
| **מזהה הפוסט** | `selectors.postIdAttrs` | האם ה-pid יושב ב-`data-pid` או ב-attribute אחר. |
| **מחבר הפוסט** | `selectors.authorUidAttrs`, `selectors.authorSlugAttrs`, `selectors.authorLink` | האם יש `data-uid` על הפוסט, ומה מבנה הקישור לפרופיל (`/user/<slug>`). |
| **מצב "כבר עשיתי לייק"** | `selectors.likedClassHints` + `aria-pressed` | אחרי לייק, בדקו איזה class נוסף לכפתור (למשל `upvoted`). זה מה שמבדיל increment מ-decrement. |
| **מזהה הנושא (topicId)** | אוטומטי | נלקח מ-`ajaxify.data.tid` (דרך ה-probe) ומה-URL `/topic/<tid>`. בדרך כלל לא צריך לגעת. |

אם NodeBB לא חושף `window.app`/`ajaxify` (גרסה/תבנית שונה), בדקו ב-Console:
`window.app && window.app.user` ו-`window.ajaxify && window.ajaxify.data`.

## אבטחה ופרטיות
- הרשאות: `storage` + `cookies` + `host_permissions` ל-mitmachim.top ולשרת
  ה-API בלבד. אין `tabs`, `history`.
- הרשאת `cookies` משמשת אך ורק לקריאת ה-session cookie (`express.sid`) של
  mitmachim.top, שמועבר לשרת שלך כדי לאמת את זהות הפורום מול NodeBB
  (`/api/self`). כך ה-uid מאומת על-ידי הפורום עצמו ולא נסמך על הלקוח. ה-cookie
  נשלח רק לשרת ה-API שהוגדר, ב-HTTPS, ולא נשמר (רק hash שלו לקאש קצר).
- אין סודות בקוד; טוקן ExtSync הוא אופציונלי (fallback) ונשמר ב-`chrome.storage.local`.
- ה-probe קורא רק נתוני משתמש ציבוריים מ-NodeBB ולא משנה דבר בעמוד.

ראו [`INTEGRATION.he.md`](INTEGRATION.he.md) לחיבור מול מודול השרת ולפרטי ה-API.
